Our Philosophy:

Government spending is the most direct window a citizen has into the government's actual priorities

Data on Government Spending Should Be:

  • Timely
  • Complete
  • Comprehensive
  • Descriptive
  • Interoperable

Timely Data

  • Ongoing trends can be detected, analyzed
  • Detection of fraud is actionable
  • Predictable release schedule
  • Newsworthy
Timely data allows for trends in the data to be detected in real time. For a business analytics firm, this might mean a custom market analysis for their customers. For NGOS and other governmental service providers, they might adjust their resources to better complement federal action. Or for journalists it could mean there's something to report on that's relevant to the public.

Complete Data

“Transparency isn’t transparency if you cannot understand the information being presented on a government website.”
-Michael Wood, Recovery Board ED

Complete Data

  • Comes with a robust data dictionary!
  • Required fields are reliably present
  • Missing data are documented

Comprehensive

  • Default mode of spending data should be disclosure.
  • Exceptions should be minimal and intuitive
    • The whole point of spending disclosure is a data driven view of how the government functions and what it's priorities are. By only releasing certain information, you lessen that value of that more and more.

Descriptive

  • Category codes helpful, but not a replacement for detailed description of spending
  • Context is important!
  • Metadata helps with context, allows for aggregation
A common use case is to look up a contract for a particular product. This is not possible with NAICS or PSC, but can probably only be found by a keyword search. Since the full text of contracts is not available, it's even more important to have a prose description that details exactly what an obligation is for.

Interoperable

  • Data can reference other data (denormalized), but the reference data must also be accessible
  • Leverage existing standards
  • spending categories, geographic locations, treasury account symbols     --> all have a standard format
It's fine for data to be denormalized, but users should be able to access the cross-referenced data, hopefully in the same fashion as the original data. Main Challenges and Barriers to Using USASpending Data

  • Incorrect or Lack Of Documentation
  • Errors, gaps in the data
  • Lack of information about data quality and compliance
  • Inability to reliably resolve entities using DUNS
  • Support for further questions, information

The biggest issue users cited is the lack of complete documentation for all of the downloadable data. Many pointed out inaccuracies in the current data dictionary as well as differences between the FPDS dictionary and the USASpending dictionary.

Poor data quality was also a big challenge for users. When compared to the CFFR, multiple users reported that USASpending had only 70-80% of what was in the CFFR for the last year the CFFR was produced. Setting aside data quality itself, most users suggested that reporting data quality or compliance metrics would help them manage expectations with their products and give them more confidence in the data.

Aggregating data by entity was a very common task that everyone wanted to be able to do. Based on the current data, this isn't possible to do accurately via the data or website.

Most users have in-depth questions about the data that are not answered by the site's FAQ. The lack of contact information or response from the site contact form has been very frustrating for them

Consumers of this information can also be producers

Many users requested a way to feed information back in to USASpending.gov, such as reporting errors or submitting fixes. They are definitely vested in making it better and more usable. Sunlight alone has built an infrastructure to normalize all of the values in the data. They can also help with disseminationg information about this data, creating tutorials on using it, or publicizing it's uses.

The Proposal:

Connect the users and producers of this data

I propose that the GAT board is in a good position to convene these kinds of users and the producers of the spending data in one place to address some of these challenges. Since the GAT board is responsible for strategic direction, it can set a precedent by engaging the community in the same way the Recovery board has in the past. It could be framed as a meeting, mini-conference or a development sprint where attendees work together to create documentation, tutorials, primers and tools for the data.

Structure

  • Roundtable discussion
  • After hours social meetups
  • Day-long mini-conference
  • Development/Documentation sprint

There are a lot of variations this could take. It could be an ongoing series, like a set of social meetups or roundtables. It could be a one time event like a mini conference that kicks off a broader discussion that can take place online. Or it could be a bit more collaborative like a development sprint.

In software development, people sprint for days on documentation alone. This community has a good enough working knowledge of the data that it could be a great resource for helping to create and distribute documentation, tutorials and guides for using this data.This community wants to help itself, but they need some level of access to the experts.

Potential Outcomes

  • Knowledge sharing with those most likely to disseminate
  • Community driven documentation
  • Guides, tutorials, data dictionaries on using the data and the site search
  • Tools for users and the USASpending team
  • The beginning of an online home for the federal spending data community
  • More informed discussions of spending transparency priorities
Here's a list of some possibilities that could results from this. Documentation that is driven by and serves the community. Collaborative guides on how to use the data. Tools for users. Maybe you have some programmers who attend who write a script to parse all the address fields and convert them to lat/long. Or maybe they create a Q&A site like Stack Exchange to host discussion about the data.

Potential Discussion Questions

  • How to link contractors to subcontractors
  • What are differences b/w USASpending and FPDS
  • How to increase public interest in spending transparency
  • How users can submit data cleaning/fixes back to USASpending

Now, I know that sometimes the requests of the transparency community can be a little all over the map. But the self identified data geeks I surveyed have very specific, practical questions that relate directly to their application. Answers to these kinds of questions are very achievable and go a long way towards building trust and credibility with this community so everyone is operating in good faith as we work toward a common goal.
